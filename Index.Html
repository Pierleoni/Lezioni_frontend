Differenze tra Html, Css e Javascript


HTML, CSS, JavaScript sono linguaggi di programmazione web, cioè codici che servono a sviluppare pagine web.
Conoscere questi tre linguaggi base è fondamentale per Web developers e per chi si occuppa di web marketing.
Quindi HTML, CSS e JavaScript servono a programmare applicazioni web accessibili e fruibili attraverso un network come la rete internet in un'architettura di tipo client-server(Un sistema basato su un archittetura di rete formata da due tipi di moduli:

  • il client
  • il server
    • che generalmente sono eseguiti su macchine diverse collegate in rete).
Quindi possiamo dire che HTML, CSS e JavaScript sono tre dei principali linguaggi del web lato client. Moltissime pagine web vengono create con questi codici. Sono molto diffusi anche perché sono abbastanza semplici e versatili.
Tra questi tre il codice Html è quello piu importante perchè costituisce la base sui cui si svillupano CSS e JS, tuttavia JavaScript è un vero e prorio linguaggio di programmazione mentre Html e CSS sono linguaggi di markup, cioe formattatori di testo perchè utilizzano i cosidetti marcatori (< e>) per caratterizzare detemrinate parole chiave o anche detti Tag.
Andiamo a vederli ennlo specifico partendo prorio da Html:
  1. HTML(Hyper Text Mark-Up Language) : Questo linguaggio, ad oggi nella versione 5, definisce il significato(la struttura semantica) di una pagina web, cioè il contenuto effettivo della pagina.
    Quindi viene usato per descrivivere i documenti e i contenuti ipertestuali, ovvero testuali e non, disponibili nel Web. Tradotto in parole povere attraverso il linguaggio Html si crea il testo scritto, elenchi puntati,immaggini, link e video.
    N.B. Tuttavia non si può assegnare uno stile grafico,fatta eccezzione per il text formatting(cioe il testo in grasetto, corsivo, sottolineato, evidenziato & Co.), difatti l'Html non si occuppa dello stile e della parte grafica.

    2. Curiosità Storiche


    Il linguaggio Html fu creato da Sir Tim Bernes-Lee nel 1991 ma fu rilasciato alle masse solo nel 1993, fu allora che usci l'HTML1 il cui scopo iniziale era quello di condividere i dati attraverso i browser web. Tuttavia la crescita di questo linguaggio si arresto poco dopo il suo rilascio poichè all'epoca non erano molte le persone che sviluppavano siti web.
    Nel 1995 fu rilasciato l'HTML2 con ulteriori caratteristiche, che lo resero il linguaggio di markup standard per il web design fino al 1997.
    Nello stesso periodo Dave Raggett svilluppò l'HTML3: offriva ai webmaster carratteristiche più performanti nella progettazione delle pagine, tuttavia le caratteristiche di quet'ultimo rallentavano i browser poichè non erano in grado di applicarle.
    Nell 1999 fu svilluppato l'HTML4.01 che fu dotato del supporto per le ozioni multimediali ed inoltre, questo aggiornamento, includeva anche fogli di stile, funzioni di stampa e linguaggi di scripting aggiuntivi.
    Nel 2015 fu rilasciato l'HTML5 in cui è stato aggiunto un maggiore supporto per l'archiviazione multimediale ed elementi di contenuto specifici. Anche il doctype inline, l'incorporamento di audio e video sono stati semplificati.
    Nel 2022 L'hTML5 viene aggiornato alla versione 5.2 che migliora la politica di sicurezza dei contenuti. Inoltre, si concentra sull'eCommerce, aggiungendo un'API per le richieste di pagamento. Questo aggiornamento ha anche aggiunto applicazioni internet accessibili e ricche di contenuti per le persone disabili.

  2. CSS (Cascading Style Sheets) : Se il linguaggio HTml definisce l'ipertesto della pagina web, CSS definisce lo stile e la parte grafica del sito.
    Ergo questo linguaggio viene usato per definire la formattazzione di documenti Html(siti web e relative pagine), quindi lo si usa per curare lo stile assegnando i colori, il design, il layout e il posizionamento degli elementi all'interno di una pagina Web o delle pagine di un sito. Ma si determinano anche le variazioni di visualizzazioni e dimensioni per i vari dispositivi.
    Quindi i CSS sono un insieme di modificatori di prorietà come dimensione, posizione, colore ed etc.
    Ogni modificatore viene applicato a specifici elementi della pagina web, identificati attraverso l'uso dei selettori. Ad esempio: .commento { width: 100px; }

    Curiosità Storiche


    L'ideazone di CSS si lega in maniera diretta con Htlm; in qualche modo è un complemento all'idea di separare il contenuto dalla presentazione. All'epoca, come ad oggi, HTML è un linguaggio strutturale senza informazioni grafiche o di stile sulla presentazione del documento, di consequenza era necessario avere un linguaggio che si occuppasse dell'aspetto visuale della pagina.
    Nel 1996 nascono ufficcialmente i CSS level 1 (CSS1) anche se la loro realizzazione risale già al 1993.
    Nel 1998 viene rilasciata la versione 2 di CSS la quale conteneva anche prorpietà per il posizionamento degli elementi in una pagina e la possibilità di definire stili per altri media (stampanti, dispositivi manuali e uditivi), oltre a un approfondimento delle proprietà già presenti.
    NEl 2004 la CSS level 2 ha la prima revisione (diventa CSS2.1) che contiene alcuni aggiornamenti rispetto alla versione precedente facendolo diventare una Recommendation definitiva nel 2011.
    Ad oggi la versione usata è i CSS level 3 (CSS3)

  3. JavaScript : JavaScript è un linguaggio di scripting orientato agli oggetti e comunemente usato nei siti web perché permette di integrare le varie pagine con link, mappe e altri elementi che si aggiornano automaticamente. Viene utilizzato per rendere dinamiche le pagine che altrimenti sarebbero statiche: stabilisce quindi il comportamento delle pagine a seconda dell'azione da parte dell'utente, come il click di un pulsante, la compilazione di form, eccetera. Come CSS, anche JavaScript fa riferimento al linguaggio HTML.

    4. Curiosità Storiche


    Javascript fu ideato da Brendan Eich nel 1995 quando lavorava per la Netscape Communications Corporation, e all'inizio chiama questo nuovo linguaggio Livescript per via della sua dinamicità e vivacità.
    All'epoca questa compagnia deteneva il primato come inventore di un browser utilizzato da tutti gli utenti in rete (Netscape) e voleva dotarlo di di un linguaggio di scripting che permettesse ai web designer di interagire con i diversi oggetti della pagina (immagini, form, link, ecc.), ma soprattutto con le applet Java (programmi che permettono di interagire con l'utente).
    Sempre nello stesso periodo la Netscape Communications Corporation aveva stretto una partnership con Microsystem (ideatrice di java) cosi il 4 dicembre 1995 entrambe le compagnie rilasciarono questo linguaggio, per il browser Netscape 2.0, descrivendolo come "complementare a HTML e a Java"(Dopo un po la Netscape cambio nome di Livescript in JavaScript in onore di Java).
    La Microsoft, loro rivale sul mercato, rispose in due modi:

    1. con l'introduzione di VBScript all'interno di Internet Explorer 3
    2. con una propria versione di JavaScript, sotto molti aspetti simile all'originale, chiamata JScript (siamo nel luglio 1996)supportata per Microsoft Explorer, causando cosi problemi di compatibilità tra i browser.
      3. A causa di queste differenze nel novembre del 1996, la Netscape e Sun si affidarno all'ECMA (European Computer Manufacturers Association) che standardizzo Javascript nel 1997 e nel 1998 la standardizzazzione venne accettata anche da ISO (International Organization for Standardization).

Fundametals

Su visual code per far in modo che il software capisca che stiamo scrivendo un codice HTML5 dobbiamo dichiararlo attraverso la dichiarazione -!DOCTYPE html-.
Tutti i documenti HTML5 devono inizare con questa dichiarazione che serve per identificare la versione del linguaggio HTML che stiamo utilizzando. Quindi grazie a questo tag è possibile capire subito con quale tipo di documento si ha a che fare, nel caso manchi dal documento si possono verificare errori nella presentazione del sito web.
In questa dichiarazione viene indicato quale linguaggio e quale versione sono utilizzati nel codice successivo, ergo definisce il tipo di documento e quindi indica la specifica utilizzata, individuando cosi quali elementi, attributi possono essere presenti.
In sostanza il doctype permette a un dispositivo di identificare le regole usate per scrivere il documento e di visuallizzarle appropriamente.
Difatti il doctype non è una caratteristica unica ed esclusiva di HTML ma riguarda anche altri linguaggi (XHTML, MathML, SVG ed XML). Nel caso specifico del linguaggio che interessa a noi, il doctype indica al browser che una pagina web è scritta secondo le regole HTML e di conseguenza indica al browser come effettuare il rendering in maniera corretta.
Rispetto alle versioni precedenti di HTML il doctype non è più basato su SMGL, percio non richiede maiuscole o minuscole.
Questa dichiarazione deve essere collocata al di fuori del tag -html-, nella prima riga del documento HTML.
!!!N.B. la dichiarazione non è un tag HTML ma è un informazione per il browser su quale tipo di documento aspettarsi. !!!
il DOCTYPE viene supportato da piu browser che sono:

  • Google Chrome (Windows, Mac, Linux, Android)
  • Microsoft Edge (Windows 10; 8; 8.1; 7, MacOS 10.12 & Later, Linux)
  • Firefox (Android, iOS 8.2, Linux, Windows 10;11 )
  • Safari(iOS, iPadOS, macOS)
  • Android

Ricapitolando questo tipo di dichiarazione ha più funzioni:
  1. Deve essere scritto nella prima riga dell'editor
  2. Deve essere collocata al di fuori del tag -html-
  3. definisce il tipo di documento con cui si sta lavorando ed il tipo di linguaggio usato
  4. Permette a un dispositivo di identificare le regole usate per scrivere il documento e di visuallizzarle appropriamente
  5. tutti i documenti Html devono iniziare con questa dichiarazione per essere conforme agli standard Html di base
  6. fa sapere al browser web del visitatore che si tratta di un documeto html5.

Curiosità Tecniche


I tag sono parole chiave che ci consentono di scrivere un documento Html organizzandolo in una struttura logico-gerarchica per organizzare la visualizzazione dei contentuti ipertestuali sullo schermo del browser.
I tag non verrano mai visualizzati dall'utente sul prorio monitor, ma potrà vedere i loro effetti poicè il browser li interpreta e li apllica. Vengono messi tra le parentesi angolari (es: <>) e ne esistono di 2 tipi:

  • Tag di coppia: Solitamente vengono forniti in coppia; uno di apertura (es: <>) e uno di chiusura(es:). La loro funzione viene applicata a tutto cio che è compreso tra l'apertura e la chiusura. Il tag di chiusura è caraterrizato dalla presenza di una sbarra che precede il nome del tag stesso.
    N.B. per ragioni di retrocompatibile è oppurtuno anteporre uno spazio prima della sbarra.
  • Tag Unici: questi tag consentono di effettuare particolari operazioni sul documento HTML che non richiedono la delimitazione del testo; ad esempio con il tag br è sufficiente indicare in quale punto del testo lo si voglia mandare accappo.

    Per cui, in questo ed altri casi, il/i tag sono unici è del tutto inutile averne uno di chiusura.

    • Curiosità Tecniche


    Come già detto alcuni tag vanno in coppia, quindi In pratica i tag sono dei contenitori di un testo che sta al loro interno.
    L'insieme di tag iniziale, contenuto e tag finale è detto elemento HTML. La loro sintassi è la seguente:
    Questo tipo di elemento, con un tag iniziale (o di apertura), del contenuto e un tag finale (o di chiusura), costituisce il tipo più comune di elemento e viene detto elemento contenitore.
    Quindi gli elementi sono identificati dai tag nel sorgente del testo. Un tag è costituito dal nome dell'elemento (di solito l'abbreviazione di un nome più descrittivo) tra parentesi angolari (<>). Il browser sa che qualsiasi testo tra parentesi va nascosto e non visualizzato nella finestra del browser.
    Quindi il nome dell'elemento viene visualizzato nel tag di apertura (detto start tag) e di nuovo nel tag di chiusura (o end tag) preceduto dallo slash (/).
    Il tag di chiusura funziona qualcosa come un interruttore “off” per l'elemento. N.B.Porre molta attenzione a non utilizzare il carattere simile, cioè il backslash, (\) nel tag finale.
    I tag aggiunti intorno ai contenuti sono indicati come markup. Tuttavia non tutti gli elementi viaggiano in coppia e non tutti gli elementi hanno un contentuto, alcuni sono vuoti per definizione, come già accenato nello specchietto riguardante i tag appena sopra (ad esmepio il tag img che è quello per inserire un immagine o il tag br). Questi elementi vuoti (detti anche void nella temrinologia W3C o standalone o empty in alcuni casi ) non hanno contenuto al loro interno: non formattano del conentuto ma servono solo a dare una direttiva. Come detto sopra non vanno in coppia ma sono elementi singoli: non vi è nessuna possbilita di inserivi un testo.

        name="viewport" content="width=device-width, initial-scale=1.0"
        Questo metadata da le instruzioni al browser su come controllare le dimesioni e il ridimensionamento della pagina.
        witdh=device-width setta la larghezza della pagina per seguire la larghezza degli screen dei vari dispositivi.
        initial-scale= 1.0 setta il livello dello zoom iniziale quando la pagina è in carica dal browser.
